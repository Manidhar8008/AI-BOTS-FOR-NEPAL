# AI-BOTS-FOR-NEPAL
We propose a step-by-step, zero-to-one plan to build a bilingual (Nepali+English) AI chatbot on a Nepali municipal website (e.g. gokarneshwornmun.gov.np) and scale it to ~200–250 such sites. The solution uses open-source tools (to minimize cost) and a Retrieval-Augmented Generation (RAG) architecture. First, we crawl and scrape all public HTML pages and PDFs from the municipal site into a document store. We then preprocess and chunk that content, extract metadata (e.g. URLs, dates, document type), and index it in a vector database. A multilingual chatbot backend (using LangChain/FastAPI) embeds user queries, retrieves relevant content, and feeds it to a local LLM (e.g. a quantized LLaMA or Intel’s neural-chat-7b via Ollama) for answer generation. Nepali queries are machine-translated to English (and answers back to Nepali) to leverage English-language models
. The system is deployed as a chat widget (e.g. a floating icon) on the website, with CI/CD pipelines to automate updates. We detail roles, timeline, tech stack options (free vs paid), costs, data schema, and governance. Deliverables include a prioritized milestone checklist (with hours and cost), comparison tables for tools, a demo script and slide deck outline, multi-tenant deployment/ops plan, and a risk registry.
